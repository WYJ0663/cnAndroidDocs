包含了高级别的类，封装了所以的 Android 应用模型。

Android 应用就是使用了 Android 四个核心应用组件中的一个或多个的应用程序。其中两个应用组件在本包中定义： {@link android.app.Activity} 和 {@link android.app.Service}。另外两个组件在 {@link android.content} 包中： {@link android.content.BroadcastReceiver} 和 {@link android.content.ContentProvider}。

{@link android.app.Activity 活动} 是用户可以与之交互，并执行一些操作的应用组件。 比如电话拨号、照相、发送电子邮件、查看地图等操作。 活动可以启动其它活动，包括其它应用中的活动。

{@link android.app.Service 服务} 是没有用户界面，可以在后台执行长时间任务的应用组件。 例如，服务可以处理网络传输、播放音乐、与内容提供器协同工作，不需要用户干预。

{@link android.app.Fragment 片段}类也是应用程序设计的重要部分— 特别是为象平板电脑这样的大屏幕设备设计程序时。片段定义活动行为的不同部分，包括其用户界面。 和活动类似，它也有自己的生命周期， 并可以与嵌入同一个活动的其它片段共存。 活动运行时，你可以添加、删除片段，包括由活动管理的后台栈中的所以片段— 允许用户退回到之前的片段状态，而不必离开活动。

该包还定义了应用程序实用工具，例如对话框、通知和动作条。

使用该包中类的更多信息，参见如下文档： 活动、 服务、 片段、 使用动作条、 创建对话框 和通知用户。
