包含创建“应用小部件”所必须的组件。用户可以将其嵌入到其它应用（比如用户桌面）中， 用以快速访问应用数据和服务，而不必启动新活动。

更多信息参见应用小部件开发者向导，

{@more}

应用小部件的行为由“应用小部件提供器”决定。“应用小部件宿主”是可以包含应用小部件的组件（比如用户桌面） 。

任何应用都可以发布应用小部件（作为应用小部件提供器）。应用要发布应用小部件时，唯一需要做的是提供一个 {@link android.content.BroadcastReceiver}，接收 {@link android.appwidget.AppWidgetManager#ACTION_APPWIDGET_UPDATE}意图，并提供应用小部件的元数据。 Android提供了扩展自{@link android.content.BroadcastReceiver}的 {@link android.appwidget.AppWidgetProvider}类，为定义应用小部件提供方便，并帮助处理广播事件。

应用小部件宿主是可以放置小部件的容器。大部分外观和感受到的细节由小部件的宿主决定。 例如用户桌面以一种方式呈现小部件；锁屏画面也可以放置小部件，它以不同的方式添加、移除和管理小部件。

关于实现应用小部件宿主的更多信息参见{@link android.appwidget.AppWidgetHost}类。
